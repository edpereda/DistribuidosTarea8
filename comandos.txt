////////////CONFIGURAR LA MAQUINA

- Crear maquina virtual (Ubuntu 18, 1gb de ram, abrir puerto 8080)

- Instalar JDK8
	sudo apt update
	sudo apt install openjdk-8-jdk-headless

- Copiar a la maquina el archivo zip tomcat

- Desempacar el archivo zip

- Eliminar el directorio "webapps" dentro de la carpeta zip y crear otro nuevo con el mismo nombre, dentro de este crear directorio "ROOT".

- Copiar a la maquina el archivo "jersey" descargado, desempacarlo con unzip y copiar todos los archivos con terminacion ".jar" de todos los directorios al directorio "lib" de Tomcat.

- Borrar el archivo javax.servlet-api-3.0.1.jar del directorio "lib" de Tomcat

- Copiar el archivo gson-2.3.1.jar al directorio "lib" de Tomcat

- Copiar el archivo mysql-connector-jav.zip a la maquina, desempaquetarlo y copiar el archivo mysql-connector...jar a "lib" de Tomcat.

///////////INICIAR EL SERVIDOR

- Definir las variables de entorno
	export CATALINA_HOME=aquí va la ruta del directorio de Tomcat 8
	export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64
- Iniciar la ejecucion con:
	sh $CATALINA_HOME/bin/catalina.sh start
- Detener la ejecucion con:
	sh $CATALINA_HOME/bin/catalina.sh stop

//////////INSTALACION DE MYSQL

- Ejecuta el comando
	sudo apt update

- Instala el paquete:
	sudo apt install mysql-server

- Ejecuta el script de seguridad:
	sudo mysql_secure_installation
  Presiona Y
  New password: password
  Re-enter password: password

  Disallow root login remotely? Y
  Remove test database and access to it? Y
  Reload privilege tables now? Y

- Ejecutar el monitor MYSQL
	sudo mysql

- Ejecutar el comando para modificar la contraseña de root
	ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'contraseña-de-root-en-mysql';

- Actualizar privilegios
	FLUSH PRIVILEGES;

- Para salir del monitor
	quit

//////////////////////////////////CREAR UN USUARIO EN MYSQL
- Ejecuta el monitor MYSQL
	mysql -u root -p

- Crea el usuario "hugo":
	create user hugo@localhost identified by 'contraseña-del-usuario-hugo';

- Otorga permisos a "hugo" sobre la base de datos servicio_web
	grant all on servicio_web.* to hugo@localhost;

- Salir modo monitor MYSQL
	quit

///////////////////////////////////CREAR LA BASE DE DATOS
- Ejecutar el monitor de MySQL (notar que ahora se utiliza el usuario "hugo"):
	mysql -u hugo -p

- Crear la base de datos "servicio_web":
	create database servicio_web;

- Conectar a la base de datos creada anteriormente:
	use servicio_web;

- Crear las tablas "usuarios" y "fotos_usuarios", así mismo, se crea una regla de integridad referencial y un índice único:

create table usuarios
(
    id_usuario integer auto_increment primary key,
    email varchar(256) not null,
    nombre varchar(100) not null,
    apellido_paterno varchar(100) not null,
    apellido_materno varchar(100),
    fecha_nacimiento date not null,
    telefono varchar(20),
    genero char(1)
);
create table fotos_usuarios
(
    id_foto integer auto_increment primary key,
    foto longblob,
    id_usuario integer not null
);
alter table fotos_usuarios add foreign key (id_usuario) references usuarios(id_usuario);
create unique index usuarios_1 on usuarios(email);

- Salir del monitor de MySQL:
	quit
